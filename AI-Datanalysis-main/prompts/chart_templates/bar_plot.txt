# BAR CHART GENERATION RULES

Create exactly ONE valid Plotly bar chart and assign it to variable: result.

## ⚠️ CRITICAL: Identify table format FIRST before writing any code

Before writing any code, look at the sample_head and column list to determine the table format:

**WIDE FORMAT** (e.g., each subject/month/category is its own column):
  - Example columns: ['StudentID', 'Toan', 'Van', 'Anh', 'Li', 'Hoa', 'Sinh']
  - Numeric columns ARE the categories the user wants to compare
  - → Use: `numeric_cols = ['Toan','Van','Anh',...]; agg = df[numeric_cols].mean(); x=agg.index; y=agg.values`
  - → Do NOT use groupby on any string column

**LONG FORMAT** (e.g., there is one column for category name and one for value):
  - Example columns: ['MonHoc', 'Diem', 'StudentID']
  - One column holds category labels, another holds the numeric value
  - → Use: `agg = df.groupby('MonHoc')['Diem'].mean(); x=agg.index; y=agg.values`

❌ NEVER plot raw rows as bars (this creates hundreds of bars = one per student/row).
❌ NEVER skip aggregation when the user asks for average/sum/count BY group.

## Libraries
Use:
- pandas as pd
- numpy as np
- plotly.graph_objects as go

## Data source
The datasets are already loaded as pandas DataFrames:
- DF_1, DF_2, DF_3, ...

Choose the most relevant DataFrame for the user request.

## Purpose of bar chart
Use a bar chart when the user wants to:
- compare values across categories
- count records by category
- compare average / sum / median / min / max by group
- show top or bottom categories
- compare one metric across provinces, products, classes, users, departments, etc.

## Critical reasoning rules
1) If the x/category column contains repeated values and the user's intent is comparison by group, you MUST aggregate before plotting.
2) Never plot raw repeated rows directly as separate bars when the user asks for:
   - average by group
   - count by group
   - sum by group
   - median by group
   - comparison by category
3) Each category should produce only ONE final bar unless the user explicitly asks for grouped bar chart or stacked bar chart.
4) If the user asks for:
   - "average by ..." -> use groupby(...).mean()
   - "count by ..." -> use value_counts() or groupby(...).size()
   - "sum by ..." -> use groupby(...).sum()
   - "median by ..." -> use groupby(...).median()
5) If there are too many categories, sort and keep top 10 or top 20 only when necessary.
6) Use the FULL relevant dataset unless the user explicitly asks for a subset such as top 50 rows, first 100 rows, or a date range.
7) Never use markdown fences. Output raw Python code only.
8) Do not call fig.show().

## Data cleaning & Filtering rules
Before plotting and aggregating:
- create a copy: df = DF_n.copy()
- CRITICAL FILTERING: If the user explicitly asks for a subset of data (e.g. "sản phẩm là laptop", "năm 2023", "chỉ lấy khu vực A"), you MUST filter the DataFrame using pandas boolean indexing (e.g., `df = df[df['product'].str.contains('laptop', case=False, na=False)]`) BEFORE applying any groupby or calculations.
- convert required numeric columns with pd.to_numeric(..., errors="coerce")
- drop rows where required x or y values are missing
- if the chart is based on aggregated values, aggregate first into a new DataFrame named agg

## Column selection rules
- x should usually be the grouping/category column
- y should usually be:
  - a numeric column, or
  - an aggregated metric such as count / mean / sum
- if x is long text, rotate labels
- if user mentions a metric explicitly, use that metric
- if user asks "average metric by X", x = X and y = mean(metric)

## Special instructions
- If the request is "average X by Y", first group by Y, then compute the average X, then plot one bar per Y.
- If the request is "count by category", plot one bar per category.
- If the request is "top N categories", sort the aggregated result and keep N rows.
- If grouped or stacked comparison is explicitly requested, create multiple go.Bar traces and set barmode="group" or barmode="stack".
- Do not generate overlapping duplicate bars for the same category.
- If the user asks for a metric across "each category" (like each subject, each month, etc.) where these categories are MULTIPLE DISTINCT NUMERIC COLUMNS:
  - DO NOT use `.groupby()` on a string column.
  - Compute the metric across the numeric columns directly (e.g. `res = df[numeric_cols].mean()`).
  - Use `res.index` as the `x` values and `res.values` as the `y` values.
